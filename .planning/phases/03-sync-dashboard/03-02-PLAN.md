const ERROR_MESSAGES: Record<SyncErrorCode, string> = {
  DUPLICATE_CODE: 'El codigo de subgrupo ya existe en el servidor. Renombra el codigo e intenta de nuevo.',
  NETWORK: 'Error de conexion. Verifica tu internet e intenta de nuevo.',
  UNKNOWN: 'Error inesperado. Intenta de nuevo.',
};

export function getErrorMessage(code: SyncErrorCode): string {
  return ERROR_MESSAGES[code];
}

Main function `syncPlantation(plantacionId, onProgress)`:
1. Refresh auth token: `await supabase.auth.getSession()` (per Pitfall 2)
2. PULL: call `pullFromServer(plantacionId)` — downloads subgroups, plantation_users, plantation_species from Supabase for this plantation and upserts into local SQLite using `db.insert().onConflictDoUpdate()`
3. Get pending: `const pending = await getFinalizadaSubGroups(plantacionId)`
4. For each pending SubGroup:
   a. Load its trees from local DB: `db.select().from(trees).where(eq(trees.subgrupoId, sg.id))`
   b. Call `onProgress({ total, completed: i, currentName: sg.nombre })`
   c. Call `uploadSubGroup(sg, sgTrees)` — wraps `supabase.rpc('sync_subgroup', {...})`
   d. Map RPC result: if `data?.success === true` -> call `markAsSincronizada(sg.id)` -> push `{success: true}` to results
   e. If `data?.success === false` -> push `{success: false, error: data.error}` (do NOT call markAsSincronizada)
   f. If RPC throws/errors -> push `{success: false, error: 'NETWORK'}`
   g. ALWAYS continue to next SubGroup (never break on failure)
5. Return all results array

Pull function `pullFromServer(plantacionId)`:
- Fetch subgroups: `supabase.from('subgroups').select('*').eq('plantation_id', plantacionId)`
- Upsert each into local SQLite using Drizzle `db.insert(subgroups).values({...}).onConflictDoUpdate({ target: subgroups.id, set: { estado: sql\`excluded.estado\`, nombre: sql\`excluded.nombre\` } })`
- Fetch plantation_users: `supabase.from('plantation_users').select('*').eq('plantation_id', plantacionId)`
- Upsert into local plantation_users table
- Fetch plantation_species: `supabase.from('plantation_species').select('*').eq('plantation_id', plantacionId)`
- Upsert into local plantation_species table
- Note: Do NOT pull trees (per Research open question 2 — trees are large, only subgroup metadata needed for dashboard)

Upload function `uploadSubGroup(sg, sgTrees)`:
- Map local field names to server field names in p_subgroup JSONB:
  - `id` -> `id`, `plantacionId` -> `plantation_id`, `nombre` -> `nombre`, `codigo` -> `codigo`, `tipo` -> `tipo`, `usuarioCreador` -> `usuario_creador`, `createdAt` -> `created_at`
- Map trees to p_trees JSONB array:
  - `id` -> `id`, `subgrupoId` -> `subgroup_id`, `especieId` -> `species_id` (null if null), `posicion` -> `posicion`, `subId` -> `sub_id`, `fotoUrl` -> `foto_url` (always null for now), `usuarioRegistro` -> `usuario_registro`, `createdAt` -> `created_at`
- Call `supabase.rpc('sync_subgroup', { p_subgroup, p_trees })`

IMPORTANT: Do NOT call `notifyDataChanged()` inside the sync loop. Call it ONCE after the entire loop in `syncPlantation` (per Pitfall 3 from research).

Create `mobile/src/hooks/useSync.ts`:

```typescript
import { useState, useCallback } from 'react';
import { syncPlantation, SyncSubGroupResult, SyncProgress } from '../services/SyncService';
import { notifyDataChanged } from '../database/liveQuery';

export type SyncState = 'idle' | 'syncing' | 'done';

export function useSync(plantacionId: string) {
  const [state, setState] = useState<SyncState>('idle');
  const [progress, setProgress] = useState<SyncProgress | null>(null);
  const [results, setResults] = useState<SyncSubGroupResult[]>([]);

  const startSync = useCallback(async () => {
    setState('syncing');
    setProgress(null);
    setResults([]);
    try {
      const res = await syncPlantation(plantacionId, setProgress);
      setResults(res);
    } catch (err) {
      console.error('Sync failed:', err);
    } finally {
      setState('done');
      notifyDataChanged();  // single refresh after all sync operations
    }
  }, [plantacionId]);

  const reset = useCallback(() => {
    setState('idle');
    setProgress(null);
    setResults([]);
  }, []);

  const hasFailures = results.some(r => !r.success);
  const successCount = results.filter(r => r.success).length;
  const failureCount = results.filter(r => !r.success).length;

  return {
    state,
    progress,
    results,
    startSync,
    reset,
    hasFailures,
    successCount,
    failureCount,
  };
}
```

Key points:
- `state` drives modal visibility: 'syncing' -> show modal, 'done' -> show results, 'idle' -> hidden
- `progress` updates in real-time during sync via onProgress callback
- `results` array available after sync completes for displaying per-SubGroup success/failure
- `reset` returns to idle state (user dismisses results)
- `notifyDataChanged()` called in finally block (exactly once, after all operations)
- `hasFailures`, `successCount`, `failureCount` computed for easy UI consumption

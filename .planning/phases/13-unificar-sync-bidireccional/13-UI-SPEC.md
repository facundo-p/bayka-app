## Spacing Scale

The project uses a named (non-standard) scale in `theme.ts`. Declared values must reference these tokens only ‚Äî no hardcoded pixel values in components.

| Token | Value | Usage |
|-------|-------|-------|
| `spacing.xs` | 4 | Icon gaps, dot positioning |
| `spacing.md` | 8 | Inner padding (badge, chip) |
| `spacing.xl` | 12 | Modal gap between elements |
| `spacing.xxl` | 16 | Section padding, card horizontal |
| `spacing.xxxl` | 20 | Modal overlay padding |
| `spacing['4xl']` | 24 | Modal inner padding |
| `spacing['5xl']` | 32 | Bottom sheet bottom padding |
| `spacing['6xl']` | 40 | Major breaks |

Exceptions:
- `spacing.sm = 6` and `spacing.lg = 10`: pre-existing tokens in `theme.ts` maintained for backward compat with existing components ‚Äî not introduced by this phase. Not multiples of 4; not used in new components defined here.
- Orange dot: 8px diameter (hardcoded in `OrangeDot` component as `size` prop default ‚Äî this is a pixel value, not a spacing token, because it is a fixed visual indicator not a layout gap)
- Count badge: 18px height (existing pattern from `PlantationDetailHeader` `countBadge` ‚Äî maintain for visual consistency)
- Touch targets: minimum 44px effective touch area via `hitSlop={8}` on small pressables (existing project pattern)

## Typography

All sizes reference `fontSize` tokens from `theme.ts`. All weights reference `fonts` tokens.

| Role | Token | Size | Weight Token | Weight | Line Height |
|------|-------|------|-------------|--------|-------------|
| Body / Label | `fontSize.base` | 15px | `fonts.regular` | 400 | 1.4 (implicit RN default) |
| Secondary label | `fontSize.sm` | 12px | `fonts.regular` | 400 | 1.4 |
| Button text | `fontSize.xl` | 16px | `fonts.semiBold` | 600 | 1.0 (button, no wrap) |
| Modal title | `fontSize.xxl` | 18px | `fonts.bold` | 700 | 1.3 |

Typography rules for this phase:
- **OrangeDot** has no typography ‚Äî it is a pure visual indicator
- **Progress text in SyncProgressModal** uses `fontSize.xl` / `fonts.regular` / `colors.textMuted` (existing `progressText` style ‚Äî do not change)
- **Phase label in modal** ("Descargando..." / "Subiendo subgrupos...") uses `fontSize.xxl` / `fonts.bold` / `colors.text` (existing `title` style ‚Äî extend, not replace)
- **Per-plantation progress line** ("Sincronizando [Lugar]... (2 de 5 plantaciones)") uses `fontSize.base` / `fonts.regular` / `colors.textSecondary` ‚Äî new style, follows existing `currentName` pattern with italic removed

## Color

All values sourced from `mobile/src/theme.ts`. No new color constants are allowed except `colors.syncPending` (new addition mandated by D-08).

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `colors.background` | `#FAFAF9` | Screen backgrounds, modal overlays base |
| Secondary (30%) | `colors.surface` | `#FFFFFF` | Cards (PlantationCard, SubGroupCard), modal body, bottom sheet |
| Accent ‚Äî pull/download actions | `colors.info` | `#2563EB` | Pull phase spinner, Descargar button (if kept as distinct state) |
| Accent ‚Äî push/primary sync actions | `colors.primary` | `#0A3760` | Unified "Sincronizar" button background, success icon, dismiss button |
| Sync pending dot | `colors.syncPending` | `#F97316` | **NEW ‚Äî orange-500.** OrangeDot background in ALL placement contexts. Must be added to `theme.ts` in the "State chips" section. |
| Destructive | `colors.danger` | `#DC2626` | Delete confirmation, error messages in SyncProgressModal |
| N/N warning | `colors.secondary` | `#99B95B` | N/N banner, partial sync warning icon |

**Accent reserved for (explicit list):**
- `colors.syncPending` (#F97316): OrangeDot on PlantationCard sidebar, OrangeDot on SubGroupCard row, OrangeDot overlay on global sync icon in PlantacionesScreen header. Used nowhere else.
- `colors.primary` (#0A3760): Unified "Sincronizar" button background, SyncProgressModal push phase spinner, success checkmark icon, dismiss button.
- `colors.info` (#2563EB): SyncProgressModal pull phase spinner only.
- `colors.stateFinalizada` (#F59E0B): Amber is already used for "finalizada" state chip. Must NOT be used for syncPending dot ‚Äî use #F97316 (orange-500) to maintain contrast distinction.
- `colors.danger` (#DC2626): Failure messages, partial sync failure icon in modal.

**Color not to use:**
- `colors.stateSincronizada` (#0A3760 ‚Äî brand blue): This was the old "sincronizado" chip color. The chip is being removed (D-10). Do not add a new green/blue "synced" dot. Absence of orange dot = synced.
- `colors.statPending` (#99B95B ‚Äî olive green): Currently used for "pending" stats. Do not use this for the sync dot. It would clash with stateActiva (same green).

## Component Inventory

### New Components

#### `OrangeDot` ‚Äî `mobile/src/components/OrangeDot.tsx`

| Property | Value |
|----------|-------|
| Default size | 8px diameter |
| Shape | Circle (`borderRadius: size / 2`) |
| Color | `colors.syncPending` from theme |
| Position | Absolute overlay, top-right corner of parent (parent must have `position: 'relative'`) |
| Accepts | `size?: number` prop (default 8) |
| Style | `StyleSheet.create` ‚Äî no inline styles |
| When rendered | Only when `pendingSync === true` for the relevant entity ‚Äî never render a "synced" version |

**Placement contexts:**

1. **PlantationCard sidebar** ‚Äî 8px dot absolute over the leaf icon (top: 0, right: 0 within `styles.sidebar`). Show when plantation has any subgroup with `pendingSync = true`.
2. **SubGroupCard row** (`PlantationDetailScreen.renderSubGroup`) ‚Äî inline 8px dot after the subgroup name, before `SubGroupStateChip`. Show when `item.pendingSync === true`.
3. **Header sync icon** (`PlantacionesScreen`) ‚Äî 8px dot absolute over the `sync-outline` Ionicons (top: 0, right: 0 within a `position: 'relative'` wrapper View). Show when any local plantation has any subgroup with `pendingSync = true`.

#### `useSyncSetting` ‚Äî `mobile/src/hooks/useSyncSetting.ts`

No visual output. Manages persistent `incluirFotos` boolean via `SecureStore`. Used by `PlantationDetailHeader` (new unified button) and optionally `PerfilScreen`.

### Modified Components

#### `PlantationDetailHeader` ‚Äî unified sync button

| Before | After |
|--------|-------|
| Two buttons: "Descargar" (info blue) + "Subir" (primary blue) | One button: "Sincronizar" (primary blue, full width) |
| Two "Incluir fotos" checkboxes | One toggle sourced from `useSyncSetting` (persistent) |
| Calls `onStartPull` + `onStartSync` separately | Calls `onStartSync(incluirFotos)` once ‚Äî hook handles pull+push |

Button spec:
- Label: "Sincronizar" with `sync-outline` Ionicons (size 18, white)
- Background: `colors.primary` (#0A3760)
- Border radius: `borderRadius.lg` (12px)
- Padding vertical: `spacing.md` (8px)
- Count badge: shows syncable count (subgroups with `estado='finalizada' AND pendingSync=true`) ‚Äî badge style same as existing `countBadge` (white bg, 18px height, primary text)
- Disabled state: `opacity: 0.5` when `syncableCount === 0 AND pendingPhotosCount === 0`
- Photo toggle: single `CheckboxRow` with label "Incluir fotos" ‚Äî reads/writes `useSyncSetting`

#### `SyncProgressModal` ‚Äî bidirectional phase display

New states added to `SyncState` type:

| State | Spinner Color | Title | Progress Text |
|-------|--------------|-------|--------------|
| `pulling` | `colors.info` (#2563EB) | "Actualizando datos..." | "Descargando novedades del servidor" |
| `pushing` | `colors.primary` (#0A3760) | "Subiendo subgrupos..." | "{completed} de {total}" + `{currentName}` in italic |
| `uploading-photos` | `colors.primary` | "Subiendo fotos..." | "{completed} de {total} fotos" |
| `downloading-photos` | `colors.info` | "Descargando fotos..." | "{completed} de {total} fotos" |
| `done` | icon only | "Sincronizacion completa" / "Sincronizacion parcial" | see existing pattern |

Global sync adds plantation-level progress:
- Below the primary progress text, show: "Sincronizando [Lugar]... ({done} de {total} plantaciones)" using `fontSize.base` / `fonts.regular` / `colors.textSecondary` ‚Äî follows existing `currentName` style pattern

**`'syncing'` state** is renamed to `'pushing'` in the type. All existing references must be updated.

#### `AdminBottomSheet` ‚Äî unified sync action

For `plantation.estado === 'activa'` section, add a "Sincronizar" `ActionItem` after "Asignar tecnicos":

```
icon: 'sync-outline'
label: 'Sincronizar'
color: colors.primary
disabled: !isOnline
onPress: onSync (new prop)
helperText: 'Sin conexion' (when offline)
```

The existing `pendingBadge` styling remains unchanged ‚Äî it already surfaces pending sync state correctly.

Remove the `plantation.estado === 'sincronizada'` section entirely (D-07 removes this state). The export actions (CSV, Excel) remain under `plantation.estado === 'finalizada'` only.

#### `PlantacionesScreen` header ‚Äî global sync button

Add a new icon button in the header row (right side, after connectivity icon):

```
icon: 'sync-outline' (Ionicons, size 18, white)
wrapper: position: 'relative', padding: spacing.xs
hitSlop: 8
onPress: handleGlobalSync
```

OrangeDot (8px) absolute over the icon when `hasAnyPending === true`.

Button is visible for both admin and tecnico roles (D-03 implies global sync is role-agnostic).

## Interaction States

### OrangeDot ‚Äî display rules

| Condition | PlantationCard dot | SubGroupCard dot | Header icon dot |
|-----------|-------------------|------------------|-----------------|
| subgroup.pendingSync = true | show (if plantation has any) | show | show (if any plantation has any) |
| subgroup.pendingSync = false, pendingPhotos > 0 | show | show | show |
| all pendingSync = false, no pending photos | hide | hide | hide |

### Unified Sincronizar button ‚Äî states

| Condition | Visual |
|-----------|--------|
| online, syncable > 0 OR pendingPhotos > 0 | enabled, count badge visible |
| online, syncable = 0, pendingPhotos = 0 | disabled (opacity 0.5), no badge |
| offline | hidden (existing pattern ‚Äî sync section hidden when `!isOnline`) |

### Global sync icon ‚Äî states

| Condition | Visual |
|-----------|--------|
| hasAnyPending = true | icon + OrangeDot overlay |
| hasAnyPending = false | icon only (no green variant ‚Äî absence = clean) |
| sync in progress | icon disabled (opacity 0.5 or press ignored) |

### SyncProgressModal ‚Äî transition flow

```
idle
  -> [user taps Sincronizar]
  -> pulling (spinner, info blue)
  -> pushing (spinner, primary blue, per-subgroup progress)
  -> uploading-photos (if incluirFotos, spinner, primary blue)
  -> done (checkmark or alert icon, result summary)
  -> [user taps Cerrar]
  -> idle
```

For global sync:
```
pulling (plantation 1 of N)
  -> pushing (plantation 1 of N)
  -> pulling (plantation 2 of N)  [iterates]
  -> pushing (plantation 2 of N)
  -> uploading-photos (all pending, if incluirFotos)
  -> done
```

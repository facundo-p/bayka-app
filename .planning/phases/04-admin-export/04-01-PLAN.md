  <action>
    Create 4 files following established project patterns (queries in src/queries/, mutations in src/repositories/, services in src/services/).

    **1. mobile/src/queries/adminQueries.ts:**
    - `checkFinalizationGate(plantacionId: string)`: Query subgroups where plantacionId matches AND estado != 'sincronizada'. Return `{ canFinalize: boolean, blocking: Array<{nombre, estado}> }`. Blocking list helps UI show which subgroups need sync.
    - `getMaxGlobalId()`: Return `MAX(global_id)` from trees table. Result is number (0 if no IDs exist). Used for seed suggestion (suggest max+1).
    - `getPlantationEstado(plantacionId: string)`: Return plantation estado string. Used to gate UI actions.
    - `getAllTechnicians(organizacionId: string)`: Query supabase (NOT local SQLite) for profiles where rol='tecnico' AND organizacion_id matches. Return array of `{id, nombre, email}`. MUST use supabase client because local SQLite has no profiles table.
    - `getPlantationSpeciesConfig(plantacionId: string)`: Query local plantationSpecies JOIN species for this plantation. Return array of `{especieId, nombre, codigo, ordenVisual}` ordered by ordenVisual ASC.
    - `getAssignedTechnicians(plantacionId: string)`: Query local plantationUsers for this plantation. Return array of `{userId, rolEnPlantacion, assignedAt}`.
    - `hasTreesForSpecies(plantacionId: string, especieId: string)`: Check if any trees in this plantation use this species. Return boolean. Used to prevent species removal when trees exist.
    - `hasIdsGenerated(plantacionId: string)`: Check if any tree in this plantation has globalId IS NOT NULL. Return boolean. Used to gate export buttons and prevent re-generation.

    **2. mobile/src/repositories/PlantationRepository.ts:**
    - `createPlantation(lugar: string, periodo: string, organizacionId: string, creadoPor: string)`: INSERT into supabase plantations table with `{ lugar, periodo, organizacion_id: organizacionId, creado_por: creadoPor, estado: 'activa' }`. Select the returned row. Then UPSERT into local SQLite plantations table (using db.insert().values().onConflictDoUpdate on plantations.id — set estado to excluded.estado). Call notifyDataChanged(). Return the created plantation data.
      CRITICAL (Pitfall 2): Do NOT rely on pullFromServer for the plantation row itself — pullFromServer only pulls subgroups/species/users. Upsert the returned row directly into local SQLite.
    - `finalizePlantation(plantacionId: string)`: UPDATE supabase plantations set estado='finalizada' where id=plantacionId. Then UPDATE local SQLite plantations set estado='finalizada' where id=plantacionId. Call notifyDataChanged().
      CRITICAL (Pitfall 6): Must update BOTH server AND local.
    - `saveSpeciesConfig(plantacionId: string, items: Array<{especieId: string, ordenVisual: number}>)`: DELETE all from supabase plantation_species where plantation_id=plantacionId. Then INSERT the new items array (map to {plantation_id, species_id, orden_visual}). Then call pullFromServer(plantacionId) to sync back (pullFromServer handles plantation_species upsert). Call notifyDataChanged().
    - `assignTechnicians(plantacionId: string, userIds: string[])`: DELETE all from supabase plantation_users where plantation_id=plantacionId. Then INSERT new rows with each userId, rol_en_plantacion='tecnico', assigned_at=new Date().toISOString(). Then call pullFromServer(plantacionId) to sync back. Call notifyDataChanged().
    - `generateIds(plantacionId: string, seed: number)`: Fetch all trees ordered by subgroups.createdAt ASC, trees.posicion ASC (JOIN trees to subgroups where subgroups.plantacionId=plantacionId). Use db.transaction() to update each tree: set plantacionId=i+1, globalId=seed+i. Call notifyDataChanged() after transaction.
      CRITICAL (Pitfall 3): ORDER BY must be deterministic — subgroups.createdAt ASC, trees.posicion ASC.

    **3. mobile/src/queries/exportQueries.ts:**
    - `getExportRows(plantacionId: string)`: SELECT trees.globalId, trees.plantacionId (as idParcial), plantations.lugar, subgroups.nombre (as subgrupoNombre), trees.subId, plantations.periodo, species.nombre (as especieNombre). JOIN trees -> subgroups -> plantations, trees -> species. WHERE subgroups.plantacionId = plantacionId. ORDER BY trees.globalId ASC. Return typed array.

    **4. mobile/src/services/ExportService.ts:**
    - `exportToCSV(plantacionId: string, plantationName: string)`: Call getExportRows. Build CSV string with header "ID Global,ID Parcial,Zona,SubGrupo,SubID,Periodo,Especie\n". Each row: quote fields that may contain commas (lugar, subgrupoNombre, periodo, especieNombre). Write to FileSystem.cacheDirectory + `${plantationName}_export.csv`. Share via Sharing.shareAsync with mimeType 'text/csv'.
    - `exportToExcel(plantacionId: string, plantationName: string)`: Call getExportRows. Use XLSX.utils.json_to_sheet with column headers 'ID Global', 'ID Parcial', 'Zona', 'SubGrupo', 'SubID', 'Periodo', 'Especie'. Create workbook, append sheet named 'Plantacion'. XLSX.write with type: 'base64', bookType: 'xlsx'. Write base64 to FileSystem.cacheDirectory + `${plantationName}_export.xlsx` using EncodingType.Base64. Share via Sharing.shareAsync with mimeType 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet'.
      CRITICAL (Pitfall 4): Always use type: 'base64' in XLSX.write — Node Buffer not available in RN.
      CRITICAL (Pitfall 5): Write to FileSystem.cacheDirectory only.

    All imports must use relative paths following project convention. Use `import { db } from '../database/client'`, `import { supabase } from '../supabase/client'`, etc.

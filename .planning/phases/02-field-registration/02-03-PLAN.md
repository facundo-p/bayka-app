Colors per locked decision (CONTEXT.md): Activa=green, Finalizada=orange, Sincronizada=blue.

```tsx
import { View, Text, StyleSheet } from 'react-native';
import type { SubGroupEstado } from '../repositories/SubGroupRepository';

interface Props {
  estado: SubGroupEstado;
}

const CHIP_CONFIG: Record<SubGroupEstado, { label: string; bg: string; text: string }> = {
  activa:       { label: 'Activa',       bg: '#e8f5e9', text: '#2d6a2d' },
  finalizada:   { label: 'Finalizada',   bg: '#fff3e0', text: '#e65100' },
  sincronizada: { label: 'Sincronizada', bg: '#e3f2fd', text: '#1565c0' },
};

export default function SubGroupStateChip({ estado }: Props) {
  const config = CHIP_CONFIG[estado];
  return (
    <View style={[styles.chip, { backgroundColor: config.bg }]}>
      <Text style={[styles.label, { color: config.text }]}>{config.label}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  chip: {
    paddingHorizontal: 10,
    paddingVertical: 4,
    borderRadius: 12,
    alignSelf: 'flex-start',
  },
  label: {
    fontSize: 12,
    fontWeight: '600',
    textTransform: 'uppercase',
  },
});
```

Step 2 — Create mobile/app/(tecnico)/plantation/[id].tsx (SubGroup list):

This is the main SubGroup management screen. It receives the plantacionId from the route params.

Rules:
- Use useSubGroupsForPlantation(plantacionId) hook for live list
- Each row shows: nombre (title), codigo (subtitle), SubGroupStateChip, N/N badge if unresolvedNN > 0
- For the N/N badge: useTrees hook is too expensive per row — instead query unresolved N/N count per subgroup OR show the badge only when subgroup is 'activa'. Simpler: show a separate "Resolver N/N" button only on the active subgroup the current user created that has N/N trees. Implement: for each activa subgroup the user created, show a "Revisar N/N" link if unresolvedNN > 0. Use a separate live query for this.
- Actually: display N/N badge on the list row by storing unresolvedNN count. Simplest approach: when a subgroup is shown, query unresolvedNN inline. This is acceptable for a small list.
- Tapping an 'activa' subgroup: navigate to plantation/subgroup/[id] (tree registration)
- Tapping a 'finalizada' or 'sincronizada' subgroup: no navigation (read-only)
- Show "Finalizar" button only on activa subgroups created by current user (canEdit check)
- Finalizar flow: Alert.alert confirmation → call finalizeSubGroup → on error show "Resolver árboles N/N antes de finalizar"
- Header: plantation lugar as title (use useLocalSearchParams to get id, then query plantation)
- FAB or header button: "+ Nuevo SubGrupo" → navigates to plantation/nuevo-subgrupo
- Pass plantacionId as route param to nuevo-subgrupo screen

Key navigation: router.push(`/(tecnico)/plantation/subgroup/${subgroup.id}?plantacionId=${plantacionId}&subgrupoCodigo=${subgroup.codigo}`)

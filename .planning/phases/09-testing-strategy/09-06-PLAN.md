6. tapOn id: "login-submit-button"

**02-register-tree.yaml** steps:
1. launchApp
2. Login steps (same as flow 01)
3. tapOn index: 0, id: "plantation-card-.*" (first plantation)

5. tapOn index: 0, id: "species-btn-.*" (first species button)
6. assertVisible: "1" (tree count)
7. tapOn id: "tree-undo-button"

**03-sync-subgroup.yaml** steps:
1. launchApp
2. Login steps (same as flow 01)
3. tapOn id: "sync-button"

Note: These are starter flows. Exact element IDs and flow steps will need tuning based on actual app state. The key value is having the infrastructure in place for the CI pipeline (Plan 01) to execute them.

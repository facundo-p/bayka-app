Use `@testing-library/react-native` `renderHook` + `act` for hook tests. Import `setOffline`/`setOnline` from `tests/helpers/networkHelper`.

Test cases:
1. "online signIn calls supabase.auth.signInWithPassword and caches credentials" — setOnline, call signIn(email, password). Verify supabase called, OfflineAuthService.cacheCredential called.
2. "offline signIn uses cached credentials via OfflineAuthService.verifyCredential" — setOffline, mock verifyCredential to return true. Call signIn. Verify no supabase call, session restored from cache.
3. "offline signIn fails when no cached credentials" — setOffline, mock verifyCredential to return false. Call signIn. Expect error state.
4. "signOut clears session but not cached tokens (for future offline login)" — call signOut. Verify session is null, but OfflineAuthService.cacheCredential NOT called with clear.
5. "authChangeListeners broadcasts signIn across instances" — render two instances of the hook. SignIn on one. Both should have session.
6. "TTL expiry rejects offline login when expired" — mock `isOfflineLoginExpired` to return true, mock OFFLINE_LOGIN_EXPIRE to true (use jest.isolateModules for this test). Call signIn offline. Expect rejection.

Note on Pitfall 3 (research): authChangeListeners bleeds between tests. Use `jest.isolateModules()` or expose `_resetListeners` in useAuth for test isolation.

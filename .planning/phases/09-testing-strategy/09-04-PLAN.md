Per ROADMAP success criterion #7: "Offline auth cycle fully tested: online login caches credentials -> offline signOut clears session (not tokens) -> offline signIn restores session from cache -> cross-instance broadcast works."

This is an integration-level test of the auth state machine. It uses real SecureStore mock (from setup.integration.ts) and real OfflineAuthService (not mocked), but mocks Supabase and NetInfo.

**Setup:**
- Mock `@react-native-community/netinfo` via networkHelper (`setOffline`/`setOnline`)
- Mock `src/database/supabase` with jest.fn() for auth methods
- Do NOT mock OfflineAuthService — use the real implementation against the SecureStore mock
- Mock `expo-secure-store` with an in-memory Map (already in setup.integration.ts)
- Use `@testing-library/react-native` `renderHook` + `act` for hook tests
- Use `jest.isolateModules` for cross-instance broadcast test to avoid listener bleed

**Test cases (full cycle, sequential):**

1. "online login caches credentials in SecureStore" — setOnline, mock supabase.auth.signInWithPassword to resolve successfully (with user data including rol, organizacion_id). Call `signIn(email, password)`. Verify OfflineAuthService.getCachedPassword(email) returns non-null (credentials were cached).

2. "offline signIn restores session from cached credentials" — setOffline. Call `signIn(email, password)` again. Verify session is restored (hook returns session with correct role), and supabase.auth.signInWithPassword was NOT called.

3. "signOut clears session but preserves cached tokens" — call `signOut()`. Verify hook session is null. Then verify OfflineAuthService.getCachedPassword(email) still returns non-null (tokens preserved for future offline login).

4. "offline signIn after signOut still works with cached credentials" — setOffline, call `signIn(email, password)`. Verify session restored again. This proves the full cycle: login -> signOut -> login offline.

5. "offline signIn fails with wrong password" — setOffline, call `signIn(email, 'wrong-password')`. Verify error state, session remains null.

6. "cross-instance broadcast propagates signIn" — render two instances of useAuth. signIn on instance 1. Verify instance 2 also has session. (Use jest.isolateModules or the internal _resetListeners if exposed.)

Note: This test exercises the REAL OfflineAuthService (salted SHA-256 hashing against SecureStore mock), making it a true integration test, not just a mock-wiring test. The unit-level useAuth tests in Plan 05 cover individual functions with full mocking; this test validates the complete cycle end-to-end.

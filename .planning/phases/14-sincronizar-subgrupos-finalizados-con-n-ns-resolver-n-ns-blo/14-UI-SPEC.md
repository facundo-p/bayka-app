## Spacing Scale

The project uses a named spacing scale from `mobile/src/theme.ts`. Values are not strictly multiples of 4 (project-specific convention). Executor must use these token names, never raw numbers.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline gaps |
| sm | 6px | Badge padding, chip vertical padding |
| md | 8px | Card row gap, element spacing |
| lg | 10px | Section margin between card elements |
| xl | 12px | Button vertical padding |
| xxl | 16px | Card padding, list content padding |
| xxxl | 20px | Section vertical spacing |
| 4xl | 24px | Modal/sheet padding, section breaks |
| 5xl | 32px | Layout gaps |

Exceptions:
- Touch targets minimum 44px height (buttons, action items) — enforced via `minHeight: 44` in `buttonPresets`
- Action items in AdminBottomSheet: `height: 48` (exceeds minimum for comfort)
- OrangeDot: 8px default (not a spacing token, it is a size)
- NN indicator badge in SubGroup card row: compact — `paddingVertical: 2, paddingHorizontal: spacing.sm` (matching StatusChip sm preset)

## Interaction Contracts

### Sync flow — N/N trees upload silently

- No warning before sync when subgroup has N/N trees
- N/N trees upload with `especieId=null`, `especieCodigo='NN'`
- No visual change to sync flow UI — same SyncProgressModal

### PlantationCard N/N stat

- Stat item appears only when `nnCount > 0`
- Tap on PlantationCard navigates to PlantationDetailScreen (unchanged)
- N/N stat is informational only — no tap target on the stat item itself

### SubGroup card N/N badge

- Badge is inline in cardRow, no additional height
- Badge disappears when all N/N are resolved (nnCount becomes 0)
- OrangeDot and N/N badge coexist when both conditions are true — they are different indicators

### Plantation finalization gate (AdminBottomSheet)

- "Finalizar plantacion" ActionItem disabled state: `opacity: 0.4` (existing pattern)
- HelperText beneath disabled item: "{X} árbol{plural} N/N sin resolver en {Y} subgrupo{plural}"
- Active state (all N/N resolved + all synced): enabled, `colors.danger` color (pre-existing irreversible action pattern)

### N/N conflict resolution (inline in NNResolutionScreen)

- Conflict banner shown above species grid when current tree has server conflict
- Conflict detected during pull (pre-push phase of bidirectional sync)
- Banner is dismissible by choosing one of the two options:
  - "Aceptar del servidor": sets local selection to server species, marks resolved, no re-sync needed for this tree
  - "Mantener la mía": dismisses banner, keeps local selection, re-sync will overwrite server with local
- Swipe navigation between N/N trees works as before — conflict banner travels with the tree

### Role permissions for N/N resolution

- Admin: can resolve any N/N in any subgroup of the plantation
- Tecnico: can only resolve N/N in subgroups they created (`usuarioCreador === userId`)
- Access gate: useNNResolution hook enforces this, UI shows resolution controls only when permitted
- Read-only view for tecnico on other users' N/N: show photo and current species (or "Sin resolver") without species selector

## States and Feedback

| State | Visual Treatment |
|-------|-----------------|
| Subgroup with N/N + pendingSync | OrangeDot (left of name) + N/N badge (right of chip) — both coexist |
| Subgroup with N/N resolved, pendingSync=true | OrangeDot only — N/N badge disappears |
| Subgroup with N/N unresolved, pendingSync=false | N/N badge only — no OrangeDot |
| Plantation with unresolved N/N | "N/N: {X}" stat in statsRow with `colors.secondaryYellowDark` icon and value |
| Plantation with all N/N resolved | N/N stat hidden (nnCount === 0 → do not render stat item) |
| Finalize button — blocked by N/N | `opacity: 0.4` + helperText below — no red highlight (same disabled pattern as sync blocker) |
| Finalize button — enabled | `colors.danger` text + icon, full opacity — same as pre-existing pattern |
| N/N conflict tree in resolution screen | Conflict banner above species grid, `colors.dangerBg` background, `colors.danger` left border (4px) |
| N/N tree read-only (other user's, tecnico role) | Species selector hidden; photo visible; "Resolución pendiente" muted label |

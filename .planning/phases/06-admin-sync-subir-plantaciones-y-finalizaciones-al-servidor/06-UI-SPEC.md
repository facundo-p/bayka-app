> Generated by gsd-ui-researcher. Source of truth for executor and ui-auditor.

| Property | Value |

## Component Inventory

New components introduced in this phase. All must import exclusively from `mobile/src/theme.ts`.

### 1. CatalogScreen (mobile/src/screens/CatalogScreen.tsx)

Full-screen shared component. Receives no props — reads `isAdmin` from `useRoutePrefix()`, `userId` from `useCurrentUserId()`, `organizacionId` from `useProfileData()`.

**Layout:**
```
SafeAreaView (background: colors.background)
  ScreenHeader
    title: "Catalogo de plantaciones"
    rightElement: null (no secondary icon needed on catalog)

  [loading state]
    ActivityIndicator (color: colors.primary, size: "large")
    centered in remaining space

  [error state]
    Ionicons "cloud-offline-outline" size=48 color=colors.textMuted
    Text (emptyTitle style): error copy
    Pressable retry button

  [empty state — no plantations on server]
    Ionicons "leaf-outline" size=48 color=colors.textMuted
    Text (emptyTitle style): empty heading copy
    Text (emptySubtext style): empty body copy

  [loaded state]
    FlatList
      contentContainerStyle: paddingHorizontal=spacing.xxl, paddingTop=spacing.xl, gap=spacing.xl, paddingBottom=96px (clears CTA button)
      renderItem: CatalogPlantationCard

    [sticky bottom bar — visible when list loaded]
      View (stickyBar style)
        Text: "{N} seleccionada(s)" OR empty string
        Pressable "Descargar seleccion"
          disabled when selectedIds.size === 0
          backgroundColor: colors.primary when enabled, colors.textDisabled when disabled
```

**State managed:**
- `catalogItems: ServerPlantation[]`
- `localIds: Set<string>`
- `selectedIds: Set<string>`
- `loadingCatalog: boolean`
- `catalogError: string | null`
- `downloadState: 'idle' | 'downloading' | 'done'`
- `downloadProgress: { total: number; completed: number; currentName: string }`
- `downloadResults: DownloadResult[]`

### 2. CatalogPlantationCard (mobile/src/components/CatalogPlantationCard.tsx)

Selectable list row. Props:
- `item: ServerPlantation`
- `isDownloaded: boolean`
- `isSelected: boolean`
- `onToggle: (id: string) => void` — no-op when isDownloaded=true

**Layout (single card):**
```
Pressable (hitSlop=8, disabled=isDownloaded)
  View (card style)
    borderLeftWidth=3
    borderLeftColor: isDownloaded → colors.stateSincronizada, else accentColor per estado

    [left: checkbox area, 44px touch target via hitSlop]
      View (28x28 checkbox)
        if isDownloaded:
          Ionicons "checkmark-circle" size=20 color=colors.stateSincronizada
          (border: none — filled icon communicates state)
        else if isSelected:
          View filled circle (backgroundColor: colors.primary, border: colors.primary)
          Ionicons "checkmark" size=14 color=colors.white inside
        else:
          View empty circle (border: 2px colors.borderMuted)

    [center: text content, flex=1]
      Text (cardTitle): item.lugar
      Text (cardSubtitle): item.periodo

      [stats row]
        Ionicons "layers-outline" size=12 color=colors.statTotal
        Text: "{item.subgroup_count} subgrupos"
        spacing.xl gap
        Ionicons "leaf-outline" size=12 color=colors.statTotal
        Text: "{item.tree_count} arboles"

      [estado chip — inline badge]
        Text chip: estado text (activa/finalizada)
        backgroundColor chip: stateActiva or stateFinalizada at 15% opacity
        Text color: corresponding state color

    [right: already-downloaded badge OR empty space]
      if isDownloaded:
        Text "Ya descargada" (fontSize.xs, color: colors.stateSincronizada, fontFamily: fonts.semiBold)
```

**Visual distinction for already-downloaded:**
- Card opacity: 0.65 (entire card dimmed)
- Checkbox replaced by filled checkmark-circle icon (colors.stateSincronizada)
- "Ya descargada" label in top-right
- No pressed state (Pressable disabled)

### 3. DownloadProgressModal (mobile/src/components/DownloadProgressModal.tsx)

Blocking modal. Structural clone of SyncProgressModal with download-specific props and copy.

Props:
```typescript
interface Props {
  state: 'idle' | 'downloading' | 'done';
  progress: { total: number; completed: number; currentName: string } | null;
  results: DownloadResult[];
  onDismiss: () => void;
}

type DownloadResult = {
  success: boolean;
  id: string;
  nombre: string;
};
```

**States:**

`idle` → returns null (not rendered)

`downloading`:
```
ActivityIndicator size="large" color=colors.primary
Text (title): "Descargando..."
Text (progressText): "{completed} de {total}"
Text (currentName, italic): "{currentName}"
```

`done` — all success:
```
Ionicons "checkmark-circle" size=48 color=colors.primary
Text (title): "Descarga completa"
Text (progressText): "{successCount} plantacion(es) descargada(s)"
Pressable "Cerrar" (dismissButton style)
```

`done` — partial failure:
```
Ionicons "alert-circle" size=48 color=colors.secondary
Text (title): "Descarga parcial"
Text successText: "{successCount} descargada(s) correctamente"
[failureSection]
  Text failureTitle: "{failureCount} con error:"
  for each failed result:
    View failureItem (colors.dangerBg background)
      Text failureName: result.nombre
      Text failureMessage: "No se pudo descargar. Reintenta desde el catalogo."
Pressable "Cerrar" (dismissButton style)
```

`done` — all failed:
```
Ionicons "alert-circle" size=48 color=colors.danger
Text (title): "Error en la descarga"
Text (progressText): "No se pudo descargar ninguna plantacion. Verifica tu conexion."
Pressable "Cerrar" (dismissButton style)
```

**Styles** (mirror SyncProgressModal exactly):
- overlay: flex=1, backgroundColor=colors.overlay, alignItems=center, justifyContent=center, padding=spacing.xxxl
- modal: backgroundColor=colors.surface, borderRadius=borderRadius.xxl, padding=spacing['4xl'], alignItems=center, width='100%', maxWidth=380, gap=spacing.xl

### 4. Modified: PlantacionesScreen connectivity icon

The existing `Ionicons` icon in `rightElement` is wrapped in a `Pressable`:

```
Pressable
  onPress: isOnline → router.push(`/${routePrefix}/plantation/catalog`)
  disabled: !isOnline
```

## Interaction Contracts

### Selection behavior (checkboxes)

- Tapping an unselected, downloadable card: adds ID to selectedIds, visual checkbox fills with colors.primary checkmark
- Tapping a selected card: removes ID from selectedIds, checkbox returns to empty circle
- Tapping an already-downloaded card: no-op — card opacity 0.65, Pressable disabled
- "Descargar seleccion" button: disabled (colors.textDisabled) when selectedIds.size === 0; enabled (colors.primary) when >= 1

### Download flow

1. User taps "Descargar seleccion" (or "Descargar {N} seleccionada(s)")
2. DownloadProgressModal appears immediately (state: 'downloading')
3. Per-plantation progress updated via callback: {completed, total, currentName}
4. On batch completion: modal transitions to 'done' state with results summary
5. `notifyDataChanged()` called once in finally block after loop
6. User taps "Cerrar": modal dismissed, screen remains on CatalogScreen with refreshed localIds (downloaded items now show as "Ya descargada")
7. selectedIds cleared on dismiss

### Navigation entry point

- PlantacionesScreen header connectivity icon (Pressable): tapping when isOnline navigates to `/(admin)/plantation/catalog` or `/(tecnico)/plantation/catalog`
- Stack route title: "Catalogo de plantaciones"
- Back navigation: standard Expo Router back arrow in stack header (headerStyle from theme.ts)

### Offline guard

- If user is offline when attempting to open catalog (e.g., connectivity changes between screens): CatalogScreen mount detects `!isOnline` and shows error state immediately without attempting Supabase fetch

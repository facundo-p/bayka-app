## Spacing Scale

All spacing values come from `mobile/src/theme.ts`. This phase uses the existing scale. No new tokens are introduced.

| Token | Value | Usage in this phase |

| xs | 4px | Icon gaps within sidebar strip |
| sm | 6px | Compact inline gaps (e.g. pending badge) |
| md | 8px | Gap between sidebar icon slots |
| lg | 10px | Internal action item padding |
| xl | 12px | Gap between action rows in bottom sheet |

## Component Inventory

New and modified components for this phase. Executor must not duplicate patterns already in the codebase.

### New: PlantationCard sidebar strip (modification to existing component)

PlantationCard gains a right-side action strip replacing the single `deleteButton` column.

| Slot | Icon | Condition | Color |
|------|------|-----------|-------|
| Top | `create-outline` (edit) | Always visible (both roles) | `colors.primary` |
| Middle | `settings-outline` (gear) | Admin only; empty slot for tecnico (no icon, same height) | `colors.primary` |
| Bottom | `trash-outline` (delete) | Always visible when `onDelete` prop present | `colors.textMuted` (idle), `colors.danger` (active) |

Strip width: 48px (matches existing `SIDEBAR_WIDTH = 48` constant).
Each slot height: 36px with `hitSlop={8}` for 44px touch target.
Vertical gap between slots: `spacing.md` (8px).
Strip background: `colors.surface` (same as card body).

### New: AdminBottomSheet component

A custom slide-up bottom sheet built with React Native `Modal` (animationType="slide") — no @gorhom/bottom-sheet.

| Property | Value |
|----------|-------|
| Modal animationType | "slide" |
| Background | `colors.surface` |
| Border radius (top corners only) | `borderRadius.xxl` (16px) |
| Drag handle | 4px tall, 40px wide, `colors.borderMuted`, centered, 12px from top |
| Backdrop | `colors.overlay` (rgba(0,0,0,0.6)) — tap backdrop to dismiss |
| Header: plantation name | `fontSize.xl` (16px), `fonts.semiBold`, `colors.textHeading` |
| Header: periodo subtitle | `fontSize.base` (15px), `fonts.regular`, `colors.textSecondary` |
| Horizontal padding | `spacing.xxl` (16px) |
| Top padding (below handle) | `spacing.xxl` (16px) |
| Bottom padding | `spacing['5xl']` (32px) + safe area inset |
| Action list gap | `spacing.xl` (12px) |
| Dismiss button (X) | Top right, `Ionicons close-outline`, size 22, `colors.textMuted` |

Action items inside the bottom sheet reuse the same `ActionItem` pattern from AdminScreen:
- Height: 48px
- Background: `colors.surfaceAlt`
- Border radius: `borderRadius.lg` (12px)
- Icon size: 18
- Label: `fontSize.base`, `fonts.medium`
- Disabled state: `opacity: 0.4` on the entire row

### Modified: PlantacionesScreen header

Admin sees two icon buttons in `rightElement`:
1. "+" button (left): `colors.primary` background, `borderRadius.full`, size 34×34. Triggers create plantation modal.
2. Download/catalog button (right): existing behavior unchanged.

Tecnico sees only the catalog button (no "+" button). No layout shift.

### Removed: AdminScreen.tsx

The entire screen is deleted. The Gestión tab is removed from `app/(admin)/_layout.tsx`. Admin and tecnico tab layouts become identical: 2 tabs (Plantaciones, Perfil).

## Interaction Contract

### PlantationCard sidebar strip

- Edit icon tap: opens `PlantationFormModal` in edit mode (existing modal, new trigger point).
  - For `activa` estado: edit allowed.
  - For `finalizada` / `sincronizada`: edit icon is shown but tapping shows a toast/helper text: "No se puede editar una plantación finalizada".
- Gear icon tap (admin only): opens `AdminBottomSheet` for that plantation.
- Trash icon tap: triggers existing delete confirmation flow (unchanged).
- All icon taps call `e.stopPropagation()` to prevent card navigation.

### AdminBottomSheet — estado-specific action sets

**Estado: activa**
1. Configurar especies — `leaf-outline`, `colors.primary`, always enabled
2. Asignar técnicos — `people-outline`, `colors.primary`, always enabled
3. Finalizar — `lock-closed-outline`:
   - Enabled (color: `colors.danger`): when all subgroups are sincronizada AND no pendingSync AND no pendingEdit
   - Disabled (color: `colors.textMuted`, opacity 0.4): when conditions not met
   - Helper text below when disabled: "Para finalizar, todos los subgrupos deben estar sincronizados" (fontSize.xs, colors.textMuted, italic)
   - If pendingSync or pendingEdit also present: show additional helper "Sincroniza los cambios antes de finalizar"

**Estado: finalizada**
- If IDs not generated:
  1. Generar IDs — `key-outline`, `colors.primary`
- If IDs generated:
  1. Exportar CSV — `document-text-outline`, `colors.primary`
  2. Exportar Excel — `grid-outline`, `colors.primary`
- Locked badge at bottom: `lock-closed` icon (12px), `colors.stateFinalizada`, label "Bloqueada", `secondaryBg` background (existing pattern from AdminScreen)

**Estado: sincronizada**
1. Exportar CSV — `document-text-outline`, `colors.primary`
2. Exportar Excel — `grid-outline`, `colors.primary`

### Bottom sheet dismiss behavior

- Tap backdrop: dismiss
- Tap X button: dismiss
- Swipe down: dismiss (Modal slide animation auto-handles)
- Any action completion: dismiss bottom sheet, then show result feedback (existing modal/alert pattern)

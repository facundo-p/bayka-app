  <name>Task 1: Create useUserNames hook with AsyncStorage caching</name>
  <files>mobile/src/hooks/useUserNames.ts</files>
  <action>
Create a hook `useUserNames(userIds: string[]): Record<string, string>` that:

1. Accepts an array of unique user UUIDs
2. Returns a map of userId -> displayName
3. Caching strategy (offline-first):
   - On mount, read cached names from AsyncStorage key `user_names_cache` (JSON object: `{[userId]: nombre}`)
   - Set state immediately from cache for any matching IDs
   - If online and there are uncached IDs, query Supabase: `supabase.from('profiles').select('id, nombre').in('id', uncachedIds)`
   - Merge new results into cache, write back to AsyncStorage
   - Update state with merged map
4. Helper function `getInitials(nombre: string): string` — exported, takes full name, returns first letter of each word (max 2 letters), uppercase. Example: "Juan Perez" -> "JP", "Maria" -> "M"
5. Helper function `getDisplayName(nombre: string, maxLength: number = 15): string` — exported, returns nombre if <= maxLength, otherwise returns getInitials(nombre). This is what the UI will use.

Use `import AsyncStorage from '@react-native-async-storage/async-storage'` (check if installed, if not use expo-secure-store pattern like useProfileData.ts but with SecureStore).

Import supabase from '../supabase/client'. Use the same offline-resilient pattern as useProfileData.ts (try/catch around Supabase call, cache-first).

Memoize the userIds array comparison with JSON.stringify to avoid re-fetching on every render.

<objective>
Decouple offline login from Supabase JWT token validity. Create a configurable TTL system where offline login expiration is controlled by app config (boolean toggle + hours), not by Supabase access_token expiry. Store last online login timestamp in SecureStore and validate against it.

Purpose: Currently offline login fails when Supabase JWT expires (default 1hr). Field workers need offline login to work for days/weeks.
Output: Config file, updated OfflineAuthService with TTL functions, updated useAuth with independent offline login flow.
</objective>

Otherwise compare: `Date.now() - storedTimestamp > OFFLINE_LOGIN_DURATION_HS * 3600 * 1000`. Return true if expired.
   - Both functions exported.
